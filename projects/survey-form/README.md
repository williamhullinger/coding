# Survey Form – Final Exam Project

## Purpose
This project was completed as a final exam for the HTML section.
The goal was to build a survey-style page that collects user input
using the correct semantic container.

The key challenge in this project was recognizing that the problem
required a **form-based solution**, not a table-based one.

## Core Lesson Learned
When a page is designed to:
- collect user input
- submit data
- include text fields, radio buttons, checkboxes, or dropdowns

👉 the correct root element is **<form>**, not <table>.

This project reinforced the importance of identifying **intent**
before writing any HTML.

## Why This Project Matters
This project represents a shift from:
- thinking in **individual elements** (tables, rows, inputs)
to
- thinking in **page purpose and structure**

It highlights that HTML problems often test **concept recognition**
rather than syntax memorization.

## When to Use This as a Reference
Use this project when you need examples of:
- Survey forms
- User input collection
- Grouping related inputs
- Accessibility-friendly labels
- Choosing <form> over <table>

## Reflection
Initially, this project felt discouraging because the correct
starting element was not obvious. After seeing the solution,
the structure became clear.

This project is saved intentionally as a reminder that:
confusion at the start does NOT mean lack of understanding —
it means the brain is learning how to identify patterns.
