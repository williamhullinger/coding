# Event Hub

## Purpose
This project demonstrates the use of **semantic HTML** to organize
content for an events-based webpage. The page separates upcoming
and past events using clear structural elements.

This project helped reinforce understanding of **sectioning content**
and when to use the <article> element.

## Core Lesson Learned
When content:
- can stand alone
- could be reused or syndicated
- represents an individual item (like an event, post, or entry)

it should be wrapped in an **<article>** element.

Sections group related content.
Articles represent independent pieces of content within those sections.

## HTML Elements Used

- nav

- section
- article

## Concepts Demonstrated
- Semantic page layout (header → main → sections)
- Navigation with internal anchor links
- Grouping related content with section
- Independent content blocks with article
- Proper heading hierarchy
- Basic image usage with alt text
- HTML comments for self-documentation

## Page Structure Overview
- Header  
  - Page title
  - Navigation links
- Main  
  - Upcoming Events section
    - Individual events as articles
  - Past Events section
    - Individual events as articles with images

## Why This Project Matters
This project marked a turning point where:
- structure felt intuitive
- element choice became clearer
- less external referencing was needed

It helped solidify the difference between:
- layout containers (<section>)
- content containers (<article>)

## When to Use This as a Reference
Use this project when you need examples of:

- Proper use of article vs section
