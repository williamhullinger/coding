# Design Brief Template

A comprehensive design brief system for web projects that supports both:
- client-friendly project intake, and
- developer-focused build planning.

This template is designed to be used as:
- a fillable web form (HTML/CSS/JS), and
- a printable PDF for offline or client use.

---

## Who This Is For
- Freelance developers
- Web designers
- Solo builders and small agencies
- Anyone who wants clearer scope, pricing, and requirements before building

---

## What’s Included
- **Client Input Sections** – business goals, audience, design direction, features
- **Developer Notes** – tech stack guidance, accessibility checklist, build planning
- **Clear Client vs Developer separation** to prevent miscommunication
- **Scalable structure** for simple sites up to full-stack apps

---

## Files
- `design-brief.md` → Full master design brief (source of truth)
- `client-brief.md` → Simplified client-facing version (coming soon)
- `/web-form` → Interactive HTML form version (in progress)

---

## Planned Features
- Multi-step client intake form
- Accessible, mobile-first layout
- Optional PDF export
- Form validation and UX improvements

## Status
🚧 Work in progress — actively being built and refined.

## Why This Exists
Most project failures come from unclear requirements and scope creep.
This template is designed to solve that problem before code is written.
