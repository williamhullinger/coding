# Interactive To-Do List (Checkbox & Link States)

This project is a practice exercise focused on HTML structure and CSS link state styling. It demonstrates how to properly nest lists, associate form inputs with labels, and style anchor elements for accessibility and interaction.

---

##  Project Overview

The page displays a to-do list where each task includes:
- A checkbox input
- An accessible label linked via `for` and `id`
- A nested list containing a resource link

CSS is used to style the different states of links, including unvisited, visited, active, and focus states.

---

##  Concepts Practiced

### HTML
- Semantic structure using `<main>`
- Unordered lists (`<ul>`) with nested lists
- Checkbox inputs with associated `<label>` elements
- Anchor (`<a>`) elements with valid `href` values
- Accessibility-friendly markup

### CSS
- Class selectors
- Pseudo-classes:
  - `:visited`
  - `:active`
  - `:focus`
- Removing default link text decoration
- Keyboard accessibility using focus outlines

---

##  Link Interaction States

The links are styled to:
- Display a custom color when unvisited
- Change color once visited
- Change color while being clicked
- Show a visible outline when focused (keyboard navigation)

This ensures both usability and accessibility.
